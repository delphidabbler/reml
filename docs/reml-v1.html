Introduction

REML is very simple markup language designed to be used to elements within a single paragraph of text.

This document provides the authoritative definition of REML v1. It also provides some notes for implementors of REML interpreters.

v1 was released on 2008-12-31.

Warning: REML v1 is obsolete and should no longer be used.

Tags

Tags are embedded within a block of plain text to describe the type of text enclosed by the tags.

The supported tags are:

  • <strong>...</strong> – The enclosed markup has strong emphasis.
  • <a href="url">...</a> – The enclosed markup is a hyper-link. The href attribute must specify the required URL, which must use the http protocol.

The following rules apply to the use of tags:

  • Tags must match. E.g. <strong> must be matched with </strong>.
  • Tags may be nested, providing the tags are balanced. E.g. <strong><a href="http://example.com">example</a></strong> is nested correctly but <strong><a href="http://example.com">example</strong></a> is invalid.

Examples: 

Make stuff <strong>stand out</strong>.
See this <a href="http://example.com">example</a>.
If an unrecognised tag is encountered in REML code then the interpreter should report an error. However, providing start and end tags are matched, the interpreter may choose to simply ignore the tags.

Character Entities

A few symbolic character entities are supported in REML. They all have analogues in XHTML or HTML.

The supported symbolic character entities are:

Character Entity Actual Character
&amp; &
&quot; "
&gt; >
&lt; <
&copy; ©

The '<' and '&' characters are special within the markup and cannot be used literally in text. The &lt; character entity must be used in place of '<' and &amp; must be used instead of '&'. For example to write x<y in REML use x&lt;y and to write you & me use you &amp; me.

To express other special symbols for which there is no symbolic character entity, numeric character entities can be used. For example to display the 'Ω' character (Unicode Greek capital letter Omega) use &#937;.

Note: Numeric entities should be used with caution because the characters they represent may vary across different text encodings, whereas symbolic entities are safe across encodings.
